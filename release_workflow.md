## Release Workflow

1. Update version in `src/dot_tasks/_version.py`.
2. Commit and push to `main`.
3. Create and push a version tag, for example:

```bash
git tag v0.1.1
git push origin v0.1.1
```

Publishing behavior:

- Pushes to `main` publish to TestPyPI (`skip-existing`).
- Tags matching `v*` publish to PyPI.
- Tags matching `v*` also create a GitHub Release with automatically generated
  release notes.
- `workflow_dispatch` is available for manual runs.

Release notes behavior:

- Notes are generated by GitHub from merged PRs/commits between releases.
- Categories and exclusions are configured in `.github/release.yml`.
