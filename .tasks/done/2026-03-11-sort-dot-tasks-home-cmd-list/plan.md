## Plan: Styled Grouped Home Command Picker

### Summary
- Refine the grouped home picker so section boundaries are more visually distinct.
- Keep the grouped structure and setup-only first-run behavior, but render section headings as explicit separator lines: `---- Inspect ----`, `---- Lifecycle ----`, etc.

### Key Changes
- Introduce a shared heading formatter for command-palette sections that renders headings as `---- {Section} ----`.
- In the arrow-key root picker:
  - keep using real `Separator` entries
  - change separator text from plain section names to the formatted separator line
  - pass an `InquirerPy` style with heading-only separator styling using `separator: "bold ansicyan"`
- In the numeric fallback:
  - keep a blank line before each section
  - print the same formatted heading line before that section’s commands
  - style the heading line only, using bold cyan when terminal styling is available and plain text otherwise
- Do not add separator text inside command description columns.
- Keep command ordering, command names, callback behavior, and setup-only first-run behavior unchanged.

### Internal Interface Changes
- Extend the command-palette rendering path in `prompt_ui.py` so headings are generated by a single formatter/helper instead of inline string assembly.
- Extend `selector_ui.select_one(...)` to accept an optional style for `InquirerPy` `select`, so the root picker can style separators without affecting other selectors.
- Keep section metadata and palette construction in `cli.py` unchanged except for using the new heading formatter.

### Test Plan
- Update prompt UI tests to assert that grouped selector options use formatted separator lines like `---- Inspect ----`.
- Add or update selector wrapper tests to verify the `separator` style is passed through to `InquirerPy select`.
- Update numeric fallback tests to assert the formatted heading lines are printed with the same section order and command numbering as before.
- Keep existing grouped-order, setup-only first-run, cancel, and command-invocation tests passing unchanged in behavior.

### Assumptions
- “Clear separation” means explicit separator-line headings plus subtle heading emphasis, not a redesign of command rows.
- The chosen heading format is exactly `---- {Section} ----`.
- The chosen emphasis is heading-only, using bold cyan for styled terminals and plain text as fallback.
- Stronger color treatment, icons, borders, or description-column separators are intentionally out of scope.
